# 记忆文档：对外分享版 Skill 初始化
**时间戳**：2026-04-09 10:56

## 问题描述
用户需要一份干净的、可以直接对外分享的 `image-to-knowledge` 技能代码。该代码不能包含用户的飞书 Token、私有模型接口 URL 等敏感数据，且默认行为应为本地化导出，而非飞书直连。

## 根因分析
用户的项目目录下的原代码已绑定了自动化流水线的参数。如果直接打包原版代码，可能会发生隐私泄漏，或他人在使用时因为缺少飞书环境而报错。

## 解决方案
从用户的个人开发目录完整克隆了一份 `image-to-knowledge`，专门放在 `skill-for-share` 归档夹中。并对关键配置文件进行脱敏与降级设置。

## 改动记录
- 从 `.codebuddy/skills/image-to-knowledge/` 复制了全量代码（含新版支持 PDF 及本地导出的功能）。
- 修改了本目录下的 `config.json`：
  - `feishu.app_token` 变更为 `"YOUR_FEISHU_APP_TOKEN"`
  - `feishu.table_id` / `view_id` 变更为 `"YOUR_TABLE_ID"` / `"YOUR_VIEW_ID"`
  - `vision.base_url` 与 `ai.base_url` 变更为 `"YOUR_LLM_BASE_URL"`
  - `pipeline.default_output_format` 设置为 `"markdown"`

## 部署记录
本项目作为开源备用代码仓库，无线上部署。分享版已处于安全可直接分发（如打 ZIP 包）的状态。
