doc-to-knowledge

将飞书文档（视频课程转录稿）通过 AI 自动提取为结构化知识点，写入飞书多维表格，供人工审核后流入下游内容流水线。

⚡ 处理流程 · 5 步

从文档读取到结构化输出的完整处理链路

1

📖 读取飞书文档 doc_reader.py

调用飞书 API 获取文档所有 Block，识别 Heading2 作为章节分隔符，将文档切分为独立章节。

输入：doc_token / URL / .docx / .md 输出：chapters = [{title, content}] feishu_client.py
2

🤖 逐章节 AI 提取 knowledge_extractor.py

对每个章节，用 System Prompt（SMB 视角策略师）+ User Prompt 模板调用 AI，提取结构化知识点数组。

输入：chapter {title, content} 输出：raw_items[] gemini_client.py / OpenAI
3

🔄 全局去重与补充 post-processing

合并所有章节的知识点，再次调用 AI 执行：去重合并 → 跨章节补充 → 统一编号 → 质量检查。

输入：all raw_items[] 输出：final_items[] (编号后)
4

📊 写入飞书多维表格 bitable_writer.py

幂等 Upsert（按知识点 ID 判重），逐条写入多维表格，自动分配序号。初始状态 = 「待审核」。

输入：final_items[] 输出：Bitable 记录 (13 字段) feishu_client.py
5

💾 本地 JSON 备份 output/

将最终知识点数据导出为本地 JSON 文件，便于调试、版本追踪和离线使用。

输入：final_items[] 输出：{doc_title}_knowledge.json

🧩 模块架构

6 个核心模块，各司其职，解耦协作

🎯 主控入口

pipeline.py

串联全流程的主入口脚本，支持多种输入模式。

  • --doc-token / --doc-url / --local-docx / --local-md
  • --dry-run 预览模式
  • --json-only 纯导出模式
  • --model 指定 AI 模型
  • --output-meta IPC 元数据输出

🔗 飞书客户端

feishu_client.py

统一的飞书 API 客户端，合并三项核心能力。

  • get_document_blocks() — 读取 docx Block
  • search_records() — 多维表格查询
  • create_record() / update_record() — 写入
  • download_file() — 云盘文件下载
  • tenant_access_token 自动认证

📖 文档解析

doc_reader.py

支持三种文档来源的统一解析层。

  • 飞书在线文档（Blocks API 解析）
  • 本地 .docx 文件（python-docx）
  • Obsidian Markdown 笔记
  • 按 Heading2 自动切分章节

🧠 AI 提取核心

knowledge_extractor.py

知识点提取的核心逻辑，含 Prompt 工程。

  • KnowledgeItem dataclass 定义
  • SMB 视角 System Prompt
  • 逐章提取 + 全局后处理两阶段
  • Playbook 操作卡片模式
  • JSON 解析容错 & 自动修复

📊 表格写入

bitable_writer.py

飞书多维表格写入模块，保障数据一致性。

  • 幂等 Upsert（知识点 ID 判重）
  • 自动序号分配
  • 限频保护（≥0.4s 间隔）
  • 批量写入（≤50 条/批）

✨ Gemini 客户端

gemini_client.py

Gemini 原生 API 的兼容层。

  • 模拟 OpenAI SDK 接口
  • 支持代理网关模式
  • 可与 OpenAI 客户端无缝切换
💡 Prompt 策略

三层 Prompt 架构确保提取质量：身份设定 → 逐章提取 → 全局精炼

🎭 System Prompt · 身份设定

你是一位数字营销领域的资深内容策略师，专门服务中小客商家 (SMB)。 【目标受众画像】 - 月预算 5千~5万的中小商家 - 痛点：预算有限、缺乏投放经验 - 需求：可直接落地的方法论 【提取原则】 1. 翻译视角：大客→中小客语言 2. 可操作性优先："怎么做" > "是什么" 3. 去术语化：行业黑话→通俗表达 4. 强调 ROI：关联投入产出比 5. 独立完整：每条不依赖其他 6. 拆到原子级：一条只讲一件事

📝 User Prompt · 逐章提取

请从以下课程转录稿章节中提取知识点。 【文档标题】：{doc_title} 【章节标题】：{chapter_title} 【章节内容】： --- {chapter_content} --- 每个知识点按 JSON 格式输出： { "title": "15字以内精炼标题", "type": "concept | how_to | pitfall | ...", "content": "200-500字完整描述", "key_points": ["要点1", "要点2"], "pain_tags": ["预算有限", ...], "difficulty": "入门 | 进阶 | 高级", ... }

🔄 Post-Processing · 全局精炼

你刚才从多个章节中分别提取了知识点， 现在需要做全局后处理： 1. 去重：合并内容高度相似的知识点 2. 补充：检查跨章节遗漏的知识点 3. 编号：{doc_title}_K{01,02,...} 4. 质量检查：独立完整 + 面向中小客 输出最终的去重、补充、编号后的 知识点 JSON 数组。

💎 知识点样例

第一课_K03 小预算也能测出爆品素材 how_to
预算有限的中小商家，不需要一次性投入大量广告费来测试素材效果。用「小额多组」策略：准备 5-8 组不同素材，每组设定 50-100 元日预算，跑 3 天数据。重点看 CTR（点击率）和转化成本两个指标，把 CTR 最高且转化成本最低的 2-3 组素材保留，其余关停。这样总共花费不超过 2000 元，就能筛选出最适合你产品的广告素材。
预算有限 缺乏经验 素材匮乏 入门
第一课_K07 出价过高的三个危险信号 pitfall
很多新手商家在投放初期容易「出价过高」导致预算快速消耗。如果你发现：① 广告预算在上午就花完了，② 获客成本是行业均值的 2 倍以上，③ 曝光量很大但转化率极低——这三个信号说明你的出价策略需要调整。建议降低出价 20-30%，同时缩小定向范围，让广告找到更精准的人群。
预算有限 不懂数据 入门
✨ 核心特性

为什么选择 doc-to-knowledge

🤖
AI 驱动提取

支持 GPT-4o / Gemini 等大模型，自动从转录稿中拆解知识点

👤
SMB 视角翻译

自动将专业术语"翻译"为中小客商家能理解的语言

幂等安全

按知识点 ID 判重，重复运行不会产生重复记录

多源输入

🎭
双模式提取

transcript 模式（课程转录）+ playbook 模式（操作卡片）

🛡️
容错解析

AI 返回的 JSON 可能不规范，内置容错 + 自动修复 + 重试机制

📊
飞书深度集成

直接读取飞书文档 Block、写入多维表格、支持批量处理

💾
双重输出

飞书多维表格 + 本地 JSON 备份，数据不丢失

🚀 快速开始

五步启动，即刻体验

复制配置模板
cp .env.example .env && cp config.example.json config.json
编辑 .env，填入飞书应用凭证和 AI API Key
→ 参考 references/SETUP_GUIDE.md 创建飞书自建应用
编辑 config.json，填入多维表格 app_token 和 table_id
安装依赖
pip3 install -r requirements.txt
运行提取
python3 scripts/pipeline.py --doc-token YOUR_TOKEN --dry-run
→ 先用 --dry-run 预览，确认无误后去掉该参数正式写入飞书
